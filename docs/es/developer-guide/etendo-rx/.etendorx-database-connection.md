## Conexión a la base de datos de EtendoRX

Cuando el usuario necesita desplegar EtendoRX en producción o existe la necesidad de probar en entornos especiales, hay disponible una amplia gama de parámetros para ajustar las conexiones a la base de datos según necesidades específicas.

### Uso básico

Para configurar la conexión a la base de datos con el [método de configuración predeterminado](https://en/technical-documentation/etendo-environment/platform/etendorx-config-server) en EtendoRX, es necesario modificar los parámetros especificados en los archivos de la siguiente manera:

#### Base de datos PostgreSQL

En el archivo gradle.properties:

```
bbdd.rdbms=POSTGRE
bbdd.driver=org.postgresql.Driver
bbdd.url=jdbc:postgresql://localhost\:5432
bbdd.sid=etendo
bbdd.systemUser=postgres
bbdd.systemPassword=syspass
bbdd.user=tad
bbdd.password=tad
bbdd.sessionConfig=select update_dateFormat('DD-MM-YYYY')
```

En la ruta /rxconfig, edite el archivo das.yaml:

```
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/etendo
    username: tad
    password: tad
```

#### Base de datos Oracle

En el archivo gradle.properties:

```
bbdd.rdbms=ORACLE
bbdd.driver=oracle.jdbc.driver.OracleDriver
bbdd.url=jdbc:oracle:thin:@localhost\:1521\:orclsid
bbdd.sid=orclsid
bbdd.systemUser=sys as sysdba
bbdd.systemPassword=oraclepassword
bbdd.user=C\#\#TAD
bbdd.password=tad
bbdd.sessionConfig=ALTER SESSION SET NLS_DATE_FORMAT='DD-MM-YYYY' NLS_NUMERIC_CHARACTERS='.,'
```

En la ruta /rxconfig, edite el archivo das.yaml:

```
spring:
  datasource:
    url: jdbc:oracle:thin:@//localhost:1521/orclsid
    username: C##TAD
    password: tad
    driver-class-name: oracle.jdbc.OracleDriver
```

### Configuración avanzada

#### Fuente de datos

De forma predeterminada y como se recomienda, utilice [HakiriCP](https://github.com/brettwooldridge/HikariCP) como biblioteca de conexión a la base de datos en EtendoRX.

> Hay disponibles varios resultados de benchmarks para comparar el rendimiento de HikariCP con otros frameworks de pool de conexiones, como c3p0, dbcp2, tomcat y vibur. Por ejemplo, el equipo de HikariCP publicó los siguientes benchmarks:
>
> HikariCP-bench-2.6.0  
> El framework es tan rápido porque se han aplicado las siguientes técnicas:
>
> Ingeniería a nivel de bytecode – se ha realizado una ingeniería extrema a nivel de bytecode (incluyendo codificación nativa a nivel de ensamblador)  
> Micro-optimizaciones – aunque apenas medibles, estas optimizaciones combinadas incrementan el rendimiento general  
> Uso inteligente del framework de Collections – el ArrayList<Statement> fue reemplazado por una clase personalizada, FastList, que elimina la comprobación de rangos y realiza escaneos de eliminación de principio a fin
>
!!! info
     Fuente: https://www.baeldung.com/hikaricp

El ajuste de la conexión puede realizarse cambiando los parámetros disponibles:

https://docs.spring.io/spring-boot/docs/current/reference/html/application-properties.html#application-properties.data.spring.datasource.hikari

#### Registro detallado

Si se necesita una salida de logs más detallada, añada al archivo das.yaml:

```
spring:
  jpa:
    show-sql: true
    properties:
      hibernate:
        format_sql: true
  
logging:
  level:
    org:
      hibernate:
        type:
          descriptor: ## Shows each hibernate SQL query
            sql: WARN

```

Esta configuración hará que el log sea más detallado para facilitar la resolución de problemas.
