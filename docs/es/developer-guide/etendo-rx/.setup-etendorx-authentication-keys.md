## Visión general
Esta página explica cómo actualizar la clave pública y la clave privada necesarias para que Etendo RX autentique las conexiones.

## Configurar las claves de autenticación de Etendo Rx
1. Abra un terminal en el directorio `etendo_rx_compose`.
2. Genere las claves pública y privada que se utilizarán en los servicios Auth y Edge:

Generación de clave privada

```plaintext
  openssl genrsa -out main/resources/private.pem 2048
```

Generación de clave pública

```plaintext
  openssl rsa -in main/resources/private.pem -pubout -outform PEM -out main/resources/public_key.pem
```

Generación de clave privada con formato PCKS8

```plaintext
 openssl pkcs8 -topk8 -inform PEM -in main/resources/private.pem -out main/resources/private_key.pem -nocrypt
```

3. Actualice el contenido de `main/rxconfig/application.yaml` copiando el contenido de la clave del archivo `main/resources/public_key.pem`.

4. Actualice el contenido de `main/rxconfig/auth.yaml` copiando el contenido de la clave del archivo `main/resources/private.pem`. Mantenga la tabulación.

5. Cree un commit local y el servidor de configuración se actualizará al reiniciar los servicios de Etendo Rx.
````
cd /main/rxconfig
git add .
git commit -m "Claves pública y privada actualizadas localmente"
````

6. Reinicie Etendo RX:
````
docker-compose -p rx -f main/docker-compose.yaml restart
````
