## Visión general

Esta página explica cómo configurar Debezium y escuchar una tabla y campos específicos en el entorno Etendo Rx.

## Cómo escuchar nuevas tablas en Etendo RX - UI de Debezium

!!! info
    Para seguir esta guía, debe tener un entorno EtendoRx con los servicios de Debezium y Kafka en ejecución. Para más información, lea [EtendoRx](https://en/technical-documentation/etendo-environment/platform/EtendoRx).  

1. **UI de Debezium**: Esta aplicación ofrece una interfaz de usuario para configurar todas las propiedades que necesitamos para escuchar las tablas que necesitamos. Con Docker iniciado, accedemos a [_http://localhost:8095/_](http://localhost:8095/).

2. Para crear un nuevo conector, haga clic en el botón Create a connector.

![](../../assets/drive/aJzitIXqJbubWW8GARdErE46voV8KjmGcf2v00hQBmGUrTbhL81J4GoeyoTFhhBZ4xiIOrQiP6Yhd99gOZbg8sga4l54BE11ssioAvRldAq-ViG43SMV2WoYEj4AZT77cZL_J2LTIoKyZYeAtg.png)

3. Seleccione PostgreSQL y haga clic en el botón Next.

![](../../assets/drive/AMPZS3Vby4NP9i40khwsXUzOa2jXcCJRIxBqR4B5Q39Tu4ELWRu2TJXA4Gr28nvYinpCIll-v-hcDVeJSQ2khtmOQ6kzziXKyM-yDe94UdkSQWd5KUl2T71elJDy_Tfzdfi7kDxckVezqZdeWw.png)

4. Una configuración básica necesita lo siguiente:

![](../../assets/drive/iGY8RsjNAaEK6NsWSqaAE7DtFfFO-h3HOTNi14cniswQJcB3upduzM7iHtIkhD2_Wl5J13KpiBSWXhraX_nubdQg0EsaKXpHSMlXWfVUM67eTnFBLTPrPKUccAqRoFDp0DixbD2HHHGdauQw5Q.png)

- **Nombre del conector**: Un nombre del conector. El que quiera utilizar.
- **Espacio de nombres**: por defecto.
- **Nombre de host**: postgres. Porque así lo llamamos en el archivo docker compose.
- **Puerto**: 5432.
- **Usuario**: postgres.
- **Contraseña**: syspass.
- **Base de datos**: etendo.

5. Una vez que hayamos completado todos los campos obligatorios, necesitamos validar los datos; para ello, haga clic en el botón Validate. Deberíamos ver un mensaje de éxito como este:

![](../../assets/drive/43UN2abdTb_x4vkSjotWUAomawnIgtJp-V7c_Xiw9pkTk2AUvgQYuSXaV9OrWSj2Ni-HTpBhGZ8JkvWEmPkrdtrp0JyI6CED24iBl8nW1DCobBsP0ZXB-0wb8DOPOGzX1Qk1AGE6m_5sxoC6cw.png)

6. Haga clic en el botón Next y, en la siguiente pantalla, debemos decidir qué tablas queremos escuchar. Por defecto, se escucharán todas las tablas.

![](../../assets/drive/sNYtCQLE0xE9y3Gg-iQaNimGbG21f_EK3O-PPae8SqJ9uQ7LkeaRaaXnH1XA85grFNpSXU4YKliN9-Yv8mDjxHXjweTxaRhN10gbtd3D55-Z2ktPN8K-goEcGgncDydvFGWIX9CAKRl9GcuE2A.png)

7. Si decidimos escuchar solo algunas tablas, necesitamos buscar la tabla en el campo de entrada Table Filter y aplicar el filtro. Para este ejemplo, necesito escuchar únicamente la tabla c_bpartner_location

![debezium-setup.png](../../assets/legacy/technicaldocumentation/platform/debezium-setup.png)

8. Una vez que terminemos con esta configuración, necesitamos validar todas las configuraciones. Para ello, haga clic en el botón Review and Finish.

9. Si todo está bien, obtendremos la siguiente pantalla:

![](../../assets/drive/KN41JFvZb7YA2Er6wHiM3rcbp1r90WRmtDkMHADxhUSEknU7TR6qVb3DP1ukTrjlPRVrqVQXJJ2nhqj9861DYMHq88FRDHGGhn1X9U0by2iaejGeRjoa6cBmvLIKChTRVaYPt8BYwQ0MfPrTzw.png)

10. Por último, pulsamos el botón Finish para crear el conector.

![](../../assets/drive/gtfy5r9_ohJjCHsISAU5p0GqItRPbG-U_7Ih1h1uIC1wUxl3rJC3ataZfQEuOvtdpAEOIkTvmLxctK6aYq8_kpWNwtHIr_HwKTV7HfMzH4xGY_ffqRcCZPrdL8ZHnnVJ5Ue-wn8JJR9oadZtIw.png)
