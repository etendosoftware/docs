## Proceso asíncrono en Etendo RX

> El procesamiento asíncrono proporciona un medio para distribuir el procesamiento que requiere una aplicación entre sistemas en un entorno de intercomunicación. Sin embargo, a diferencia del procesamiento de transacciones distribuidas, el procesamiento es asíncrono.
> En el procesamiento asíncrono, el procesamiento es independiente de las sesiones en las que se envían solicitudes y se reciben respuestas. No se puede establecer una correlación directa entre una solicitud y una respuesta, y no se pueden hacer suposiciones sobre el momento en que se recibirá la respuesta.

Etendo RX permite el desarrollo de procesos asíncronos de forma simplificada. Cuando un proceso interno o externo necesita ejecutar un proceso, lo encola con una llamada POST al endpoint correspondiente con todos los datos necesarios para ejecutarlo. En primer lugar, el servicio devuelve un uuid (un carácter aleatorio con una longitud de 32) si el proceso es aceptado para su ejecución. Esta primera respuesta es solo de referencia; su ejecución aún no está confirmada.  
Internamente, se inicia el flujo necesario para esta tarea específica. Para saber cómo va y el resultado, debe realizar una llamada GET al endpoint del proceso asíncrono incluyendo el uuid de referencia.

### Módulo asyncprocess

Este módulo tiene los siguientes endpoints:

#### POST /async-process/?process={{process name}}

`process name`: El nombre del proceso a iniciar

Este endpoint le permite iniciar la ejecución de un proceso que se ejecutará más adelante.

> Este servicio requiere autenticación; para más información, siga la guía a continuación: [Módulo de autenticación de EtendoRX]()

```
POST /async-process/?process={{process name}} HTTP/1.1
Host: localhost:8099
X-TOKEN: {{token}}
Content-Type: application/json
Content-Length: 2291

{
/// Data needed to execute the process
}

```

Una vez realizada la llamada, la respuesta será:

```
{
   "id": "{{process id}}"
}
```

`process id`: uuid del proceso. Es obligatorio como referencia para conocer el estado del proceso.

#### GET /async-process/{{process id}}

`process id`: uuid del proceso.
