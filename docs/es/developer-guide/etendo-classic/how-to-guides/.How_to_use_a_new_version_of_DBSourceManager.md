#  Cómo usar una nueva versión de DBSourceManager

##  Objetivo

DBSourceManager es la herramienta que Openbravo utiliza para todas sus tareas relacionadas con la gestión de bases de datos, y está detrás de las conocidas tareas create.database, update.database, export.database y export.config.script.

En la mayoría de situaciones, su funcionalidad es suficiente para realizar cualquier tarea de desarrollo necesaria. Sin embargo, Openbravo sigue actualizándola para dar soporte a más funcionalidades de base de datos y corregir errores, y en algunos casos puede ser conveniente usar una versión más reciente para exportar los cambios de desarrollo a los módulos antes de actualizar Openbravo.

Este documento describe cómo usar el último dbsourcemanager en una versión anterior de Openbravo.

##  Cómo actualizar DBSourceManager

En primer lugar, es necesario tener acceso a la rama principal de Openbravo. Desde allí, obtendrá los archivos más recientes de dbsourcemanager. La propia rama está aquí  . Puede obtener los archivos usando Mercurial:

hg clone  https://code.openbravo.com/erp/devel/main  openbravo

El proceso es sencillo; solo necesita copiar los siguientes archivos desde la rama principal y sustituir los correspondientes de su instancia de Openbravo por ellos:

  * src-db/database/lib/dbsourcemanager.jar 
  * src-db/database/build.xml 
  * src-db/database/model/excludeFilter.xml 

Después de haber copiado los archivos, puede usar las tareas normales de base de datos (update.database, export.database, ...), y estarán utilizando el nuevo dbsourcemanager.
