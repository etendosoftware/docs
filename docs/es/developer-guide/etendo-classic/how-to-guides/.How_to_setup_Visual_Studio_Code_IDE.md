#  Configuración

Una vez que el editor esté instalado en su sistema, deben instalarse y configurarse algunas extensiones para facilitar el desarrollo.

##  ESLint y Prettier

VSCode soporta ambas herramientas instalando extensiones. Consulte [  para configurar ESLint y Prettier en su IDE  ] para más detalles.

##  Sincronizar cambios de WebContent con los archivos del módulo

Esta es una funcionalidad práctica que permite editar código javascript directamente desde la carpeta WebContent, de forma que los cambios puedan reflejarse simplemente refrescando la página, sin necesidad de ejecutar 'ant smartbuild' y relanzar Tomcat. La sincronización es bidireccional: los cambios de WebContent se copian al código original del módulo y viceversa.

Para usar esta funcionalidad, instale la extensión **Run on Save** (_emeraldwalk.runonsave_) y añada la siguiente configuración en settings.json. Para acceder a settings.json, pulse Ctrl + Shift + P en VSCode y, a continuación, busque Preferences: Open Settings (JSON).:

    
    
    ...
      "emeraldwalk.runonsave": {
        "commands": [
          {
            "match": "WebContent/web/(?!org.openbravo.userinterface.smartclient|org.openbravo.userinterface.smartclient.dev|userinterface.skin.250to300Comp)",
            "cmd": "dir=$( echo ${file} | sed -r 's#.*WebContent/web/([^/]+)/.*#\\1#') && file_mod=$(echo ${file} | sed -r 's#WebContent/web/'$dir'/#modules/'$dir'/web/'$dir'/#' ) && cp ${file} $file_mod"
          },
          {
            "match": "modules/(?!org.openbravo.userinterface.smartclient|org.openbravo.userinterface.smartclient.dev|userinterface.skin.250to300Comp)",
            "cmd": "dir=$( echo ${file} | sed -r 's#.*modules/([^/]+)/.*#\\1#') && file_mod=$(echo ${file} | sed -r 's#modules/'$dir'/web/'$dir'/#WebContent/web/'$dir'/#' ) && cp ${file} $file_mod"
          }
        ]
      }
    ...

##  Integración con PostgreSQL

Para ejecutar consultas SQL directamente en su instancia de VSCode necesita la extensión **PostgreSQL** (_ms-ossdata.vscode-postgresql_). Una vez instalada, necesita añadir la configuración de la base de datos en el archivo del espacio de trabajo (*.code-workspace):

    
    
    ...
      "settings": {
        "pgsql.connections": [
          {
            "host": "localhost",
            "dbname": "openbravo",
            "user": "tad",
            "password": "tad",
            "emptyPasswordInput": false,
            "port": "5432",
            "profileName": "openbravo_pgsql"
         }
       ],
     }
    ...

##  Ejemplo de settings.json

Este .json incluye todos los ajustes mencionados anteriormente:

    
    
    {
       // js format on save
       "[javascript]": {
           "editor.formatOnSave": true,
           "editor.defaultFormatter": "esbenp.prettier-vscode"
       },
       "prettier.requireConfig": true,   
    
       // Copy edited files from modules to webContent and from webContent to modules
       "emeraldwalk.runonsave": {
           "commands": [
               {
                   "match": "WebContent/web/(?!org.openbravo.userinterface.smartclient|org.openbravo.userinterface.smartclient.dev|userinterface.skin.250to300Comp)",
                   "cmd": "dir=$( echo ${file} | sed -r 's#.*WebContent/web/([^/]+)/.*#\\1#') && file_mod=$(echo ${file} | sed -r 's#WebContent/web/'$dir'/#modules/'$dir'/web/'$dir'/#' ) && cp ${file} $file_mod"
               },
               {
                   "match": "modules/(?!org.openbravo.userinterface.smartclient|org.openbravo.userinterface.smartclient.dev|userinterface.skin.250to300Comp)",
                   "cmd": "dir=$( echo ${file} | sed -r 's#.*modules/([^/]+)/.*#\\1#') && file_mod=$(echo ${file} | sed -r 's#modules/'$dir'/web/'$dir'/#WebContent/web/'$dir'/#' ) && cp ${file} $file_mod"
               }
           ]
       },
    
       // default vscode config
       "editor.suggestSelection": "first",
       "vsintellicode.modify.editor.suggestSelection": "automaticallyOverrodeDefaultValue",
    
       // exclude in search window the generated js files (i.e. ab7bbbd81f40d14b3b44c3241ec26a7b.js)
       "files.exclude": {
           "**/WebContent/web/js/gen": true
       }
    }
