#  Introducción

A partir de **PR19Q3** , Openbravo utiliza Prettier como formateador de JavaScript y ESLint como herramienta de linting (  más detalles  ). Entre sus ventajas, ambos están soportados por la mayoría de los IDE modernos, por lo que pueden configurarse para comprobar el linting mientras usted programa y formatear correctamente los archivos al guardarlos.

#  Eclipse

No se conocen plugins para añadir soporte para ESLint o Prettier. Sin embargo, los scripts integrados de Openbravo todavía pueden utilizarse.

#  Visual Studio Code

Visual Studio Code soporta tanto ESLint como Prettier mediante extensiones. Por favor, instale las extensiones que se muestran en la captura de pantalla siguiente:

![](/assets/developer-guide/etendo-classic/how-to-
guides/How_To_Setup_ESLint_And_Prettier_In_Your_IDE-0.png){: .legacy-image-style}

A continuación, asegúrese de ejecutar `npm ci` en la carpeta raíz de Openbravo para instalar las herramientas antes de utilizarlas en su IDE.

Para acceder a settings.json, pulse Ctrl + Shift + P en VSCode y, a continuación, busque Preferences: Open Settings (JSON).

![](/assets/developer-guide/etendo-classic/how-to-
guides/How_To_Setup_ESLint_And_Prettier_In_Your_IDE-1.png){: .legacy-image-style}

Para **Prettier** , con el fin de utilizar nuestro archivo de configuración para formatear, debe habilitarse la siguiente opción: "Prettier: Require Config", o en settings.json:

    
    
    {
    ...
      "prettier.requireConfig": true,
    ...
    }

Además, añada la siguiente línea en settings.json para habilitar el autoformateo del archivo al guardarlo:

    
    
    {
    ...
       "editor.formatOnSave": true,
    ...
    }

#  IntelliJ

IntelliJ Ultimate tiene soporte integrado para ESLint y soporta Prettier mediante un plugin.

  * **ESLint** : En Settings, habilite ESLint y defina las ubicaciones del intérprete de Node y del paquete ESLint. 

![](/assets/developer-guide/etendo-classic/how-to-
guides/How_To_Setup_ESLint_And_Prettier_In_Your_IDE-2.png){: .legacy-image-style}

  * **Prettier** : Instale el plugin Prettier de JetBrains y, en la pantalla de Settings, defina únicamente la ubicación del intérprete de Node y del paquete Prettier. A continuación, puede formatear el archivo usando Prettier con Ctrl+Alt+Shift+P o el menú contextual. 

![](/assets/developer-guide/etendo-classic/how-to-
guides/How_To_Setup_ESLint_And_Prettier_In_Your_IDE-3.png){: .legacy-image-style}

#  Sublime Text

**ESLint** funciona simplemente instalando SublimeLinter y, a continuación, los paquetes SublimeLinter-eslint usando PackageControl.

**Prettier** puede utilizarse instalando el paquete JsPrettier en PackageControl. Por defecto, este plugin buscará el CLI de Prettier en el siguiente orden:

  * Prettier instalado localmente, relativo al directorio raíz del archivo de proyecto de Sublime Text. 
  * Directorio home del usuario 
  * Directorio del plugin JsPrettier de Sublime Text 
  * Prettier instalado globalmente 

Si no aplica ninguno, es necesario introducir la ruta del CLI de Prettier en la configuración de JsPrettier (Preferences|Package Settings|JsPrettier):

    
    
    {
    ...
      "prettier_cli_path": "<path_to_prettier_executable>"
    ...
    }

#  Atom

Atom soporta tanto ESLint como Prettier mediante paquetes.
