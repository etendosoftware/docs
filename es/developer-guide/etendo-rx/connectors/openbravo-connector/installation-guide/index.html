    Instalación: Conexión de Openbravo POS y Etendo

    Visión general

    Esta guía proporciona instrucciones paso a paso para instalar y configurar la integración entre los entornos de Openbravo y Etendo. Esta configuración le permite generar documentos en el Terminal de Openbravo POS, almacenarlos en el entorno de Openbravo y sincronizarlos con Etendo utilizando Etendo RX como middleware.

    En esta guía partiremos de dos entornos limpios utilizando datos de prueba, lo que facilita la configuración y demuestra el potencial de esta integración.

    Info

    Esta guía se basa en Openbravo 23Q4.2 y Etendo Classic 24.4.0

    Software y herramientas necesarias

    • Eclipse IDE
    • Apache Tomcat
    • Docker Desktop o lazydocker
    • Postman
    • Openbravo 23Q4.2
    • Etendo Classic 24.4.0 o posterior.
    • Platform Extensions Bundle última versión.

    Openbravo

    En primer lugar, siga la Guía de instalación personalizada de Openbravo e instale en un entorno local siguiendo Cómo configurar Eclipse IDE.

    Módulos a instalar en Openbravo

    En el entorno de Openbravo es necesario instalar los módulos de Retail, EDL y Business API.

    Para obtener estos módulos, se requiere una licencia activa de Openbravo y acceso a Openbravo Forge. Asegúrese de que el entorno de Openbravo incluya los siguientes módulos en versiones compatibles con la versión correspondiente de Openbravo Core. En este ejemplo, los módulos se utilizarán en la versión 23Q4.2.

    • org.openbravo.retail.sampledata versión 23Q4.2
    • org.openbravo.retail.returns versión 23Q4.2
    • org.openbravo.retail.posterminal versión 23Q4.2
    • org.openbravo.retail.poshwmanager versión 23Q4.2
    • org.openbravo.retail.pack versión 23Q4.2
    • org.openbravo.retail.discounts versión 23Q4.2
    • org.openbravo.retail.config versión 23Q4.2
    • org.openbravo.mobile.core versión 23Q4.2
    • org.openbravo.events.core versión 1.0.233000
    • org.openbravo.api versión 1.0.234001
    • org.openbravo.service.openapi versión 1.1.233000
    • org.openbravo.service.external.integration versión 2.0.234000
    • org.openbravo.externaldata.integration versión 2.0.234000

    Warning

    Es necesario aplicar un cambio en el archivo modules/org.openbravo.api/src/org/openbravo/api/util/ApiUtils.java en la línea 38; reemplace por:

        Matcher m = Pattern.compile("(^|[^\\\\]{2}):{1}([a-zA-Z0-9_]+)").matcher(hqlWhereClause);

    Además, debe clonar en la carpeta /modules los módulos de integración de Etendo Connector. Para ello, se requiere el nivel de acceso Partner a la organización Etendo Software en GitHub:

    git clone git@github.com:etendosoftware/com.etendoerp.integration.openbravo.git --branch 1.0.0
git clone git@github.com:etendosoftware/com.etendoerp.integration.openbravo.template.git --branch 1.0.0

    Configurar el entorno de Openbravo

    Ejecute los siguientes comandos para configurar el entorno de Openbravo:

    Terminal
    ant setup
ant install.source
ant import.sample.data -Dmodule=org.openbravo.retail.sampledata
ant smartbuild

    Warning

    Configure Tomcat para que se ejecute en el puerto 8081, ya que el puerto 8080 estará ocupado por Etendo; a continuación, reinicie Tomcat.

    Etendo

    Configurar el entorno de Etendo

    Siga los pasos de la guía de instalación de Etendo, en la solapa Pasos para instalar Etendo con base de datos Postgres y Tomcat dockerizado.

    Info

    En este punto asumimos que usted ya dispone de un entorno local de Etendo, y de los servicios de Tomcat y Postgres SQL dockerizados.

    Warning

    No es necesario configurar Tomcat, ya que el servicio ya está dockerizado y preconfigurado.

    Instalar módulos

    Una vez que tenga el entorno de Etendo, debe instalar el bundle Platform Extensions y los módulos específicos del conector de Openbravo.

    Para ello, añada las dependencias en el archivo build.gradle:

    build.gradle
    dependencies {
    implementation ('com.etendoerp:platform.extensions:2.10.0') // version 2.10.0 or later

    moduleDeps('com.etendoerp:integration.to.openbravo:1.0.0@zip')
    moduleDeps('com.etendoerp:integration.to.openbravo.sampledata:1.0.0@zip')
}

    Además, en el archivo gradle.properties se deben añadir las siguientes variables de configuración, para ejecutar todos los servicios dockerizados necesarios:

    gradle.properties
    context.name=etendo

bbdd.sid=etendo
bbdd.port=5434
bbdd.systemUser=postgres
bbdd.systemPassword=syspass
bbdd.user=tad
bbdd.password=tad

org.gradle.jvmargs=-Dfile.encoding=UTF-8

docker_com.etendoerp.docker_db=true
docker_com.etendoerp.tomcat=true
docker_com.etendoerp.etendorx=true
docker_com.etendoerp.etendorx_async=true
docker_com.etendoerp.etendorx_connector=true
docker_com.etendoerp.integration.to_openbravo=true

etendorx.dependencies=com.etendorx.integration.to_openbravo:mapping:1.0.0, com.etendorx.integration.to_openbravo:worker:1.0.0
etendorx.basepackage=com.etendorx.integration.to_openbravo.mapping

    Warning

    Dado que el servicio de base de datos se ejecutará dockerizado y se asume que Openbravo está instalado en un servicio Postgres local en el puerto 5432, cambiaremos el puerto de la base de datos de Etendo a ddbb.port=5434 para que, cuando se levante el servicio Postgres dockerizado, lo haga en ese puerto.

    Ahora, en una terminal en el proyecto de Etendo, ejecutamos los comandos:

    1. Para aplicar los cambios en la configuración del archivo gradle.properties.

      Terminal
      ./gradlew setup
      2. Expandir los módulos de código fuente:

      Terminal
      ./gradlew expandModules
      3. Levantar los servicios dockerizados:

    Terminal
    ./gradlew resources.up

    Compilar y acceder a Etendo Classic

    El siguiente paso es compilar el entorno de Etendo y aplicar los datos de ejemplo.

    Terminal
    ./gradlew update.database
./gradlew import.sample.data -Dmodule=com.etendoerp.integration.to.openbravo.sampledata 
./gradlew smartbuild

    Una vez compilado el entorno, el servicio de Tomcat se reinicia automáticamente, tal y como puede verse en los últimos pasos de la compilación.

    Acceder a Etendo Classic:

    • http://localhost:8080/etendo
    • Usuario: admin
    • Contraseña: admin

    Configuraciones de Etendo RX

    Tras iniciar los servicios dockerizados, hay algunas configuraciones que deben realizarse en Etendo Classic.

    Configuración de la Entidad

    Aplicación > Configuración General > Entidad > Entidad

    Es necesario configurar el token de cifrado para la autenticación en la ventana Entidad con el rol System Administrator. Si el tiempo de expiración es igual a 0, los tokens no caducan.

    Genere una clave aleatoria con el botón Generar Clave.

    Ventana RX Config

    Aplicación > Etendo RX > RX Config

    Esta ventana de configuración almacena los datos de acceso para los servicios de Etendo RX, que son cruciales para la interacción entre diferentes servicios. Con el rol System Administrator, en esta ventana, ejecute el proceso Initialize RX Services en la barra de herramientas.

    Tras la ejecución de este proceso, se completan las variables de configuración por defecto, en función de la configuración de la instancia y de la infraestructura; incluso se configuran los parámetros por defecto requeridos por cada servicio.

    Warning

    En el caso particular del Worker Service, es necesario añadir el parámetro connector.instance, que hace referencia al ID de la instancia del conector con Openbravo. Esta instancia se distribuye dentro del conector, por lo que siempre debe establecer el ID 6364E51BF1234094A313F17E1DCD2F7D.

    Scripts de configuración inicial

    Terminal
    cd modules/com.etendoerp.integration.to.openbravo/utils
make set_wal
make insert
make setupconnector

    Relanzar los servicios de RX

    A continuación, para ejecutar efectivamente todos los servicios, es necesario ejecutar el comando en la terminal:

    Terminal
    ./gradlew resources.up

    Aquí se pueden ver todos los servicios y sus respectivos logs en ejecución usando lazydocker o Docker Desktop para una gestión de contenedores sencilla y rápida.
